5. Company logo upload is still not aligned with the architecture

Files:

client/src/components/settings/CompanySettings.jsx
client/src/api/integrations.js

This is subtle but important.

Your schema/comments suggest logo is an object storage key, but the actual upload flow still uses FileReader and stores a data URL/base64 string through UploadFile.

So:

it works
but it is not architecturally clean
and it is not using the same storage strategy as the rest of the app

For a tiny logo, this is acceptable.
For a robust business app, I would eventually make logo upload use the same proper storage flow as documents.
