Required behavior:
1. On invoice create and invoice update, recompute all money values on the server before saving:
   - lineTotal = round(quantity × unitPrice, 2)

     - ZeroRated / International / exempt / reverse_charge = 0

   - totalAmount = round(subtotal + vatAmount, 2)

2. On delivery order create and delivery order update, apply the same server-side calculation rules.

   - Ignore item.line_total
   - Ignore body.subtotal
   - Ignore body.tax_amount
   - Ignore body.total_amount

   - quantity must be a positive number
   - unit_price must be zero or positive
   - reject invalid, missing, NaN, negative, or non-numeric quantity/unit_price with HTTP 400
   - reject documents with no valid line items

5. Store only server-computed values:
   - invoice line item lineTotal must be server-computed
   - invoice amount and vatAmount must be server-computed
   - delivery order item lineTotal must be server-computed
   - delivery order subtotal, taxAmount, totalAmount, taxRate, and taxTreatment must be server-computed/resolved
